Hardware Inventory System
During my tenure at Bealls Inc., I spearheaded the development of a robust Hardware Inventory System. The primary objective was to streamline hardware tracking across the entire organization, optimizing inventory management and operational efficiency.

Key Components of the Project:
Database Infrastructure: I designed and implemented a Microsoft SQL Server database to store essential hardware information. This centralized repository allowed seamless data retrieval and updates. The database schema accommodated details such as hardware specifications, serial numbers, locations, and maintenance history.

Web Application and API: Leveraging C# and Blazor within Visual Studio, I crafted a web application that served as the user interface for managing hardware assets. The API layer facilitated communication between the frontend and the database, ensuring smooth data flow. Rigorous testing using Swagger validated the system’s functionality and reliability.

Frontend Experience: To enhance user productivity, I employed Radzen to style the frontend. The visual interface provided intuitive access to hardware data. There were different security levels for diefferent users meaning access to certain functions were limited. Users could view inventory details, track hardware movements, and initiate many data requests effortlessly even having the ability to export as spreadsheet or csv.

Greenlight for Q&A Testing: As I transitioned from Bealls, the project received approval for quality assurance (Q&A) testing.

Spotify Stats
I am developing a Python web application that interacts with the Spotify API to display information about the user’s playlists and recently played tracks. The primary objective was to provide users data about their spotify that spotify or any similiar apps did not provide.

Key Components of the Project:
Spotify API Integration: I integrated the Spotify API to fetch data about the user’s playlists and recently played tracks. This allows users to view their music data in a user-friendly interface.

Genre Analysis: I implemented a feature that analyzes the genres of the user’s recently played tracks and displays the genre distribution in a pie chart. This provides users with insights into their listening habits.

User Authentication: I used Spotify’s OAuth for user authentication. Users can log in securely using their Spotify account, ensuring that their data is protected.

Web Application Framework (Flask): The project was developed using Flask as the web application framework. Flask’s flexibility and simplicity made it an ideal choice for this project.

Web Application Server (Gunicorn) & Web Server (Nginx): I used Gunicorn as the web application server and deployed the Flask application using Nginx. This combination ensured efficient handling of requests and robust performance.
